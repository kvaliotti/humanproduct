# Positioning Review

Take existing positioning as input and run a structured audit. Diagnose weaknesses, score each component, and prescribe specific fixes.

## Step 1: Collect the Positioning

Ask the user for their current positioning. Accept any format:
- A positioning document, canvas, or statement
- A homepage or landing page URL
- A pitch deck
- A verbal description ("we're X for Y")
- A filled 5+1 canvas

If the user provides something partial (e.g., just a tagline), ask for whatever else they have. Work with what exists — don't demand perfection before starting.

## Step 2: Extract the 5+1 Components

Map whatever the user provided to the 5+1 framework. For each component, determine whether it's:
- **Explicit** — clearly stated in the positioning
- **Implicit** — can be inferred but not stated
- **Missing** — not present and not inferable

Present the extraction as a filled canvas (using your best interpretation) and ask the user to confirm or correct before proceeding.

## Step 3: Run the 9-Question Pressure Test

Use `references/pressure-test-scoring.md` for the detailed rubric.

For each of the 9 tests, score as:

| Score | Meaning |
|-------|---------|
| ✅ Pass | Positioning holds up on this dimension |
| ⚠️ Weak | Partially meets criteria but has gaps |
| ❌ Fail | Does not meet criteria — fix required |

Run these tests:
1. **Substitution test** — Could a competitor swap in their name and the positioning still sound true?
2. **10-names test** — Can you name 10 real accounts that fit the target?
3. **Alternative test** — Are non-obvious alternatives (spreadsheets, humans, nothing) included?
4. **Adjective test** — Count hype adjectives (seamless, powerful, intuitive). Target: zero.
5. **Proof test** — Does every value theme cite specific, checkable proof?
6. **Category test** — Does the market category trigger correct feature expectations?
7. **Kill-the-tagline test** — Would a salesperson know what to say with just the canvas?
8. **Sharpness test** — Can a new hire explain it in 60 seconds?
9. **Cross-functional alignment test** — Would all teams tell the same story?

## Step 4: Component-Level Diagnosis

For each of the 5+1 components, assess quality:

### Competitive Alternatives
- Are they what CUSTOMERS would actually use (not what the company thinks competitors are)?
- Are non-obvious alternatives included (Excel, manual, do nothing)?
- Are they ranked by frequency?

### Unique Attributes
- Are they provable (facts, not opinions)?
- Do they go beyond product features (delivery model, business model, expertise)?
- Are they truly unique relative to the stated alternatives?

### Value Themes
- Is there a clear Feature → Benefit → Value chain?
- Are themes clustered (1-4, not a laundry list)?
- Does each theme have proof?

### Target Market
- Is it specific enough to build a prospect list?
- Is it defined by actionable characteristics (not just demographics)?
- Could you name 10 real accounts?

### Market Category
- Does it trigger helpful assumptions?
- Were alternatives considered and rejected with reasoning?
- Is the positioning style (Head to Head / Big Fish Small Pond / New Category) appropriate for their situation?

### Trends
- If present: does it pass the three-circle test (strengths + market + customer relevance)?
- If absent: is that the right call, or is there an obvious trend being missed?

## Step 5: Deliver the Audit

Output a structured report:

**1. Overall Score** — Pass / Needs Work / Major Rework, with a 1-sentence summary.

**2. Pressure Test Results Table**
| Test | Score | Finding |
|------|-------|---------|
| 1. Substitution | ✅/⚠️/❌ | ... |
| ... | ... | ... |

**3. Component Scorecard**
| Component | Quality | Key Issue |
|-----------|---------|-----------|
| Competitive Alternatives | Strong/Weak/Missing | ... |
| ... | ... | ... |

**4. Top 3 Fixes** — Prioritized by impact. For each:
- What's wrong (specific)
- Why it matters (consequence of leaving it)
- How to fix it (concrete action, referencing the relevant step in the 10-step process)

**5. What's Working** — Acknowledge strengths. Don't make it all negative.

**6. Recommended Next Step** — Either:
- "Run the full positioning workshop to rebuild from scratch" (if Major Rework)
- "Fix these 2-3 specific components" (if Needs Work)
- "Minor polish — here are tweaks" (if Pass with notes)

## Style Rules

- Be direct. "This positioning is generic" is more useful than "there might be opportunities for improvement."
- Always provide evidence for scores. "Fails the substitution test because [competitor X] could use this same language" is useful. "Fails" alone is not.
- Don't just diagnose — prescribe. Every identified problem gets a specific fix.
- Reference case studies from `references/case-studies.md` when a relevant example exists.
