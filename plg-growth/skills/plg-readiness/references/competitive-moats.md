# Competitive Moat Assessment for PLG

## Why Moats Matter for PLG

PLG products are inherently easier to try — which means they are also easier to leave. If a competitor can replicate your free experience, users will switch. Moats are what make PLG defensible over time.

A strong moat means: the more users use your product, the harder it becomes for competitors to displace you.

---

## Moat Category 1: Network Effects

### Direct Network Effects
The product becomes more valuable as more users join the SAME side.

**Assessment questions:**
- Does the product enable communication/collaboration between users?
- Is the value proportional to the number of people using it?
- Would a single user get value, or does it require others?

**Scoring:**
| Score | Description |
|-------|-------------|
| 5 | Core value IS the network. Product is useless without other users. (Slack, Teams, WhatsApp) |
| 4 | Significantly better with more users. Solo use possible but limited. (Figma, Notion, Miro) |
| 3 | Some network benefit. Collaboration features exist but product works solo. (Google Docs, Airtable) |
| 2 | Minimal network effect. Sharing/collaboration is secondary. (Grammarly, Loom) |
| 1 | No network effect. Pure single-player product. (Calculator, text editor) |

**Building direct network effects in PLG:**
- Add real-time collaboration to core workflows
- Create shared workspaces/projects that require multiple users
- Make the product the communication layer for a workflow
- Design features that improve with team participation

### Indirect Network Effects (Platform/Marketplace)
The product becomes more valuable as more users join the OTHER side.

**Assessment questions:**
- Is there a two-sided dynamic? (Creators/consumers, buyers/sellers, developers/users)
- Does more supply attract more demand (or vice versa)?
- Is there a template/marketplace/integration ecosystem?

**Scoring:**
| Score | Description |
|-------|-------------|
| 5 | Full marketplace/platform dynamics. Both sides depend on each other. (App Store, Airbnb) |
| 4 | Strong ecosystem. Templates, plugins, or integrations by third parties. (Figma Community, Notion templates) |
| 3 | Emerging ecosystem. Some third-party content/integrations. (Zapier integrations, community templates) |
| 2 | Internal ecosystem only. Company-created templates/integrations. |
| 1 | No ecosystem. Standalone product. |

**Building indirect network effects in PLG:**
- Create a template/asset marketplace
- Open APIs and encourage third-party integrations
- Build a community platform where users share work
- Enable user-generated content that attracts new users

### Data Network Effects
The product becomes smarter/better as more data flows through it.

**Assessment questions:**
- Does the product use aggregate data to improve for all users?
- Are there AI/ML features that improve with more usage?
- Does benchmarking or comparison improve with more participants?

**Scoring:**
| Score | Description |
|-------|-------------|
| 5 | Core product IS data-driven learning. Unusable without the data. (Google Search, Waze, recommendation engines) |
| 4 | Key features powered by aggregate data. Competitive advantage from data scale. (Grammarly, Amplitude benchmarks) |
| 3 | Some data-driven features. Benchmarks, suggestions, auto-complete. |
| 2 | Data used internally but not visible to users as product value. |
| 1 | No data network effects. Product works identically with 1 or 1M users. |

**Building data network effects in PLG:**
- Add benchmarking features ("Your metrics vs. industry average")
- Build AI features that improve with usage volume
- Create recommendation engines based on aggregate patterns
- Offer industry insights derived from anonymized user data

## Moat Category 2: Switching Costs

### Data/Content Lock-in
How much user-created data, content, or configuration would be lost or costly to migrate?

**Assessment questions:**
- How much data/content do users create in the product?
- Is there an export function? How complete is it?
- How long would migration to a competitor take?
- Would historical data/insights be lost?

**Scoring:**
| Score | Description |
|-------|-------------|
| 5 | Years of irreplaceable data. Migration would take months. System of record. (Salesforce, enterprise databases) |
| 4 | Significant content/data. Migration possible but painful. Weeks of work. (Notion, Confluence, Airtable) |
| 3 | Moderate data. Migration is annoying but doable in days. (Project management tools, simple CRMs) |
| 2 | Light data. Easy to recreate elsewhere. Hours of work. (Simple tools, utilities) |
| 1 | No data lock-in. Product is stateless or data is trivially portable. |

### Workflow Integration Lock-in
How deeply embedded is the product in the user's daily workflow and tool ecosystem?

**Assessment questions:**
- How many other tools integrate with this product?
- Is the product part of a daily/weekly workflow?
- How many team members depend on it?
- Would changing tools require retraining the team?

**Scoring:**
| Score | Description |
|-------|-------------|
| 5 | Central hub of workflow. 10+ integrations. Team processes built around it. |
| 4 | Key workflow tool. 5-10 integrations. Significant process redesign to switch. |
| 3 | Regular use tool. 2-5 integrations. Some process changes to switch. |
| 2 | Peripheral tool. 0-2 integrations. Easy to replace in workflow. |
| 1 | Standalone tool. No integrations. No workflow dependency. |

### Habit / Learning Lock-in
How much has the user invested in learning the product and building habits around it?

**Assessment questions:**
- How long does it take a new user to become proficient?
- Have users customized workflows, shortcuts, or preferences?
- Is there muscle memory or keyboard shortcuts users rely on?
- Would users need to relearn a competitor's UX?

## Moat Category 3: Data Moat / Ecosystem Lock-in

### System of Record Status
Is the product the authoritative source of truth for important data?

**Assessment questions:**
- Is the product where key business data lives? (Customer records, project plans, financial data, design assets)
- Do other tools pull data FROM this product?
- Would losing access mean losing critical business information?

### Ecosystem Hub Position
Is the product a central node connecting multiple other tools and workflows?

**Assessment questions:**
- How many integrations does the average power user connect?
- Is the product the "glue" between other systems?
- Do partners/vendors build on top of this product?

## PLG-Specific Moat Implications

- **Weak moats + PLG = vulnerable.** Competitors will offer similar free experiences. Must build moat deliberately.
- **Strong network effects + PLG = flywheel.** More free users → more network value → more paid users → more free users.
- **Strong data moat + PLG = compounding advantage.** More usage → better product for everyone → harder to replicate.
- **High switching costs + PLG = retention insurance.** Even if competitor launches better free tier, your users won't switch.
