# Activation Levers

Complete checklist of levers for improving activation rate and reducing time-to-value. Use as a diagnostic: walk through each section, assess current state, identify gaps.

---

## 1. Onboarding Levers

### Welcome Experience
- [ ] **Personalized welcome:** Greeting references user's name, company, or use case
- [ ] **Use-case selection:** First screen asks what they want to accomplish (routes to relevant path)
- [ ] **Role-based path:** Different onboarding flows for different roles (admin vs. user, creator vs. consumer)
- [ ] **Progress indicator:** User can see where they are in the setup process

### Onboarding Checklist
- [ ] **3-5 items maximum:** Each item is a meaningful action (not busywork)
- [ ] **Leads to activation:** Completing the checklist = reaching the aha moment
- [ ] **Persistent but dismissible:** Visible on dashboard, can be minimized
- [ ] **Progress celebration:** Visual reward at each step and on completion
- [ ] **Links to action:** Each item deep-links to the relevant feature

### Empty State Design
- [ ] **Not actually empty:** Sample data, templates, or suggestions fill the void
- [ ] **Clear CTA:** Single primary action button ("Create your first X")
- [ ] **Value preview:** Shows what the product looks like when populated
- [ ] **Educational content:** Brief explanation of what this space is for

### Templates
- [ ] **Relevant to use case:** Template selection matches onboarding question answers
- [ ] **Pre-populated:** Templates include sample data that makes sense
- [ ] **Editable instantly:** User can modify the template in their first action
- [ ] **Varied complexity:** Quick templates (1 min) and comprehensive templates (5 min)

## 2. Engagement Levers (Post-Signup, Pre-Activation)

### Notifications / Nudges
- [ ] **Day 0 email:** Welcome + single CTA to complete first action
- [ ] **Day 1 push/email:** "Pick up where you left off" with deep link
- [ ] **Day 3 email:** Tip relevant to their use case
- [ ] **Day 7 email:** Social proof (what similar users achieved)
- [ ] **In-app nudge:** If user returns but has not activated, show targeted prompt

### Tips and Guidance
- [ ] **Contextual tooltips:** On first encounter with each feature
- [ ] **"Did you know" prompts:** Surface underused features relevant to user's use case
- [ ] **Best practice suggestions:** "Most users start by doing X"
- [ ] **Inline help:** ? icons or help panels that explain features without leaving the page

### Milestones
- [ ] **First action celebration:** Acknowledge the first meaningful action
- [ ] **Progress milestones:** "You have completed 3 of 5 setup steps"
- [ ] **Value milestones:** "You have saved X hours this week" (once data exists)
- [ ] **Team milestones:** "Your team has Y active members" (for collaborative products)

---

## 3. Education Levers

### In-Product Education
- [ ] **Product tour:** 3-5 step interactive walkthrough on first login
- [ ] **Feature spotlights:** Highlight new or relevant features with brief explanations
- [ ] **Interactive tutorials:** Step-by-step guided tasks within the product
- [ ] **Sandbox mode:** Safe space to experiment without affecting real data

### External Education
- [ ] **Getting started video:** 60-90 seconds, embedded in product
- [ ] **Use case guides:** Written guides for specific JTBDs
- [ ] **Knowledge base / Help center:** Searchable, well-organized docs
- [ ] **Community access:** Link to community where users help each other
- [ ] **Webinars / Office hours:** Regular sessions for new users (for enterprise)

---

## 4. Friction Removal Levers

### Steps Eliminated
- [ ] **Audit: count steps from signup to aha.** Target: < 5 meaningful steps.
- [ ] **Defer non-essential steps:** Move setup tasks to after first value delivery
- [ ] **Combine steps:** Can two screens become one? Can a form be shorter?
- [ ] **Auto-detect:** Infer settings from context (timezone, language, company size from domain)

### Defaults
- [ ] **Smart defaults for all settings:** User should not need to configure to get started
- [ ] **Pre-built workspace:** Default project/workspace ready to use
- [ ] **Pre-connected integrations:** If possible, auto-connect common tools
- [ ] **Sensible notification defaults:** On by default for important, off for noise

### Authentication
- [ ] **SSO / OAuth:** Google, Microsoft, GitHub (match your ICP's tools)
- [ ] **Magic link option:** Passwordless for return visits
- [ ] **Domain auto-join:** Company members auto-join existing workspace
- [ ] **Remember session:** Long session duration to reduce re-auth friction

---

## 5. Human Touch Levers

### Automated Human Outreach
- [ ] **Welcome email from a person:** CEO/PM name, reply-to works, personal tone
- [ ] **Triggered outreach at friction:** If user is stuck > 10 min, offer chat
- [ ] **Onboarding check-in:** Email or in-app message at day 3 asking if they need help

### Live Support
- [ ] **In-app chat:** Available during onboarding, < 5 min response time
- [ ] **Book-a-call option:** For users who prefer guided setup
- [ ] **Concierge onboarding:** For enterprise accounts (auto-detect from email domain)

### Community
- [ ] **New user channel:** Dedicated space for onboarding questions
- [ ] **Peer support:** Connect new users with experienced users
- [ ] **User showcases:** Examples of what activated users have built

---

## 6. Measurement Levers

### Core Metrics
- [ ] **Activation rate:** % of signups who reach the activation event within time window
- [ ] **Time-to-value (TTV):** Median time from signup to activation event
- [ ] **Onboarding completion rate:** % who complete the onboarding checklist
- [ ] **Step-by-step conversion:** Funnel from signup to each onboarding step to activation

### Segmented Metrics
- [ ] **Activation rate by channel:** Which acquisition sources produce users who activate?
- [ ] **Activation rate by use case:** Which JTBDs activate fastest?
- [ ] **Activation rate by cohort:** Is activation improving over time?
- [ ] **TTV by segment:** Which segments need more help?

### Leading Indicators
- [ ] **First session duration:** Longer first sessions (to a point) correlate with activation
- [ ] **First session actions:** How many meaningful actions in session 1?
- [ ] **Return visit rate:** % of signups who come back within 24 hours
- [ ] **Onboarding checklist engagement:** % who interact with at least one item

### Lagging Validation
- [ ] **Activated vs. non-activated retention:** 2x+ difference confirms metric validity
- [ ] **Activated vs. non-activated monetization:** Higher conversion to paid confirms metric
- [ ] **Activated vs. non-activated NPS:** Higher satisfaction confirms metric
